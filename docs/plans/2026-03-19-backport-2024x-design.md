# Backport from magicdraw-metacrawler-2024x

Date: 2026-03-19
Branch: claude/epic-easley

## Goal

Port improvements made in the 2024x fork back into the 2022x project, adapting where necessary to retain 2022x library compatibility.

## Changes

### MetacrawlerService.java

Append `[HumanType]` to target element labels so users can see the UML element type alongside the label in the context menu.

```java
// line 49 — before
String targetLabel = RepresentationTextCreator.getRepresentedText(target);

// after
String targetLabel = RepresentationTextCreator.getRepresentedText(target)
        + " [" + target.getHumanType() + "]";
```

### pom.xml

- Bump version: `1.0-SNAPSHOT` → `1.0.0`
- Replace `magicdraw.install.dir` property with parameterized `md.path`, `md.lib.path`, `md.version`, `md.build` (values: 2022.2.0, 168-7c289b26)
- Clean groupIds/artifactIds to match 2024x conventions (system-scope labels only; actual JAR paths unchanged)
- Retain `common_md` dependency (2022x-specific; no equivalent in 2024x)
- Retain EMF versions 2.38.0 / 2.40.0 (as shipped with 2022x)
- Add explicit `<source>11</source><target>11</target>` to maven-compiler-plugin configuration

### build_dist.sh

- `VERSION="1.0"` → `"1.0.0"`
- `JAR_NAME` → `"magicdraw-metacrawler-$VERSION.jar"` (was hardcoded)
- Remove static `cp "src/main/resources/plugin.xml"` copy
- Add step 4: generate `plugin.xml` dynamically with correct `$VERSION` and `$JAR_NAME`
- Renumber steps 4→5, 5→6, 6→7

### src/main/resources/plugin.xml

Delete — now generated by `build_dist.sh`.

## Out of Scope

- MagicDraw library version (stays at 2022xR2 HF3)
- Any 2024x API changes (not applicable to 2022x)
